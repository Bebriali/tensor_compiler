# 1. Исходный код (Source)

Исследуемая функция выполняет арифметические операции над элементами массива в зависимости от **четности индекса**.

```c
int functional(int *a, int n) {
    int result = 0;
    for (int i = 0; i < n; ++i) {
        int x = 2;
        int y = 1;

        if (i % 2 == 0)
            result += a[i] * x * y;
        else
            result -= a[i] + x * y;
    }
    return result;
}
```

# 2. Примененные оптимизации

## 1. Свертка констант

Компилятор обнаруживает, что
**было**
```
x = 2
y = 1
```
**стало:**

```
x * y → 2
```

Таким образом:

- выражение вычисляется **на этапе компиляции**
- лишние инструкции умножения **удаляются из тела цикла**

---

## 2. Strength Reduction (Снижение стоимости)
**Суть:**
Замена дорогих операций на более дешевые битовые.

**Результат:**

| Исходная операция | Оптимизированная |
|---|---|
| `i % 2 == 0` | `i & 1 == 0` |
| `val * 2` | `val << 1` |

## 3. Loop Unrolling (Развертка цикла)

**Тип:** Цикловая оптимизация

**Суть:**
Цикл разворачивается так, чтобы **обрабатывать два элемента за итерацию**:

- `i`
- `i + 1`

**Результат:**

- убирается условное ветвление `if (i % 2)`
- порядок **четный → нечетный** становится частью структуры цикла
- уменьшается количество переходов

<details>
<summary> Нажмите, чтобы развернуть код Generic IR</summary>

```rust
bb0:
    res0 = 0
    i0 = 0
    br bb1

bb1: ; Loop Header
    i1 = phi(i0, i_inc)
    res1 = phi(res0, res_inc)
    ; Проверка, есть ли еще хотя бы 2 элемента
    limit = n - 1
    cmp = i1 < limit
    br cmp, bb_body, bb_remainder

bb_body: ; Развернутое тело
    ; Итерация i (Четная)
    val_e = load a[i1]
    term_e = val_e << 1
    res_e = res1 + term_e

    ; Итерация i+1 (Нечетная)
    i_next = i1 + 1
    val_o = load a[i_next]
    term_o = val_o + 2
    res_o = res_e - term_o

    i_inc = i1 + 2          // Шаг сразу через один
    res_inc = res_o
    br bb1

bb_remainder: ; Обработка последнего элемента, если n нечетное
    last_cmp = i1 < n
    if (last_cmp) {
        val_last = load a[i1]
        res_final = res1 + (val_last << 1)
    }
    br bb_exit
```
</details>

# 3. Финальное оптимизированное SSA IR

<details>
<summary>Итоговый граф после всех трансформаций (включая расстановку φ-функций):</summary>

```rust
bb0: ; Вход
    res0 = 0
    i0 = 0
    br bb1

bb1: ; Заголовок цикла
    i1 = phi(i0, i_inc)
    res1 = phi(res0, res_inc)
    if (i1 < n) goto bb_body else goto bb_exit

bb_body: ; Тело (развернуто на 2 итерации)
    ; Обработка четного i
    val_e = load a[i1]
    res_e = res1 + (val_e << 1)

    ; Обработка нечетного i+1
    i_next = i1 + 1
    val_o = load a[i_next]
    res_o = res_e - (val_o + 2)

    i_inc = i1 + 2
    res_inc = res_o
    br bb1

bb_exit: ; Выход
    return res1
```

# 4. Анализ сложности

## Временная сложность

### Исходный код

Количество операций пропорционально размеру массива:

```
T(n) = O(n)
```

Каждая итерация выполняет:

- проверку `%`
- условное ветвление
- несколько арифметических операций

---

### После оптимизаций

Алгоритмическая сложность **не изменилась**:

```
T(n) = O(n)
```

Однако:

- уменьшилось число инструкций
- уменьшилось число ветвлений
- уменьшилась стоимость арифметических операций

Также благодаря **Loop Unrolling**:

- число итераций цикла ≈ `n / 2`

---

## Пространственная сложность

Алгоритм использует **константное количество переменных**:

```
S(n) = O(1)
```
